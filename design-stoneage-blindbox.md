## Problem Statement

做一个公开上线的网页版抽盲盒程序，主题是石器时代宠物图鉴收集。用户每天登录抽取盲盒，获得随机宠物图鉴卡，构建自己的收藏集。

## What Makes This Cool

- **3D盲盒旋转爆破** — 一个 3D 纹理盒子在空中旋转，点击后炸裂，宠物 GIF 直接显现
- **稀有度差异视觉** — 普通（绿）/稀有（蓝）/传说（金）各有不同的爆炸特效和粒子颜色
- **怀旧IP** — 石器时代宠物，情怀拉满
- **零延迟开盒** — 爆炸到宠物出现是一个连续物理事件，没有过渡屏

## Constraints

- 纯前端 + 轻量后端（Python FastAPI + SQLite）
- 宠物数据统一放后端
- 单人项目，MVP 不搞付费

## Premises

1. 宠物数据统一由后端管理，前端通过 API 获取
2. 用户系统：用户名 + 密码注册/登录
3. 每日免费 3 次，次数存后端，UTC 时间自然日重置
4. MVP 纯免费，不做付费
5. 先做网页版（响应式），不考虑小程序/App
6. 前端使用 Three.js 做 3D 盒子和粒子效果
7. 开盒流程：Three.js 渲染 3D 爆炸 → 切换到 HTML/CSS 层展示 GIF（混合渲染）
8. 稀有度概率：普通 60% / 稀有 30% / 传说 10%
9. 重复宠物：计数堆叠（记录每只获得次数）
10. 不使用 GSAP，Three.js 原生动画 + CSS Animation/Transition 完成时间线

## GIF 渲染策略：混合渲染

Three.js 不原生支持 GIF 动图纹理。采用混合渲染方案：

1. **Three.js 层**：负责 3D 盒子的建模、自转动画、点击爆炸、粒子飞散
2. **HTML/CSS 层**：爆炸完成后，3D 渲染区上方叠加 HTML 层展示 GIF，用 CSS 动画控制出现时机和特效（弹性缩放、光晕、稀有度边框等）
3. **衔接**：爆炸粒子的消失和 GIF 的出现严格同步，视觉上无缝衔接

**优点：** 无需 GIF 拆帧或视频转码，浏览器原生支持 GIF 渲染，开发复杂度最低。

## Visual Effects

### 开盒动画流程
1. 3D 盒子浮在场景中，缓慢自转 + 轻微上下浮动
2. 用户点击盒子 → 盒子加速旋转（~0.5s）
3. 盒子顶点向外爆裂 → 200-400 粒子飞散（Three.js 粒子系统）
4. 粒子在 t+0.8s 开始淡出
5. t+1.0s：HTML 层宠物 GIF 弹性缩放（CSS `scale(0)→scale(1)`）从爆炸中心出现
6. 根据稀有度附加特效：
   - **普通（绿）**：棕色石质粒子，绿色光晕边框
   - **稀有（蓝）**：蓝色粒子 + 微光 + 蓝色脉冲边框
   - **传说（金）**：金色粒子 ×2 + 屏幕震动（CSS）+ 金色闪光边框 + 粒子拖尾

### 整体视觉风格
- 深色渐变背景凸显发光特效
- 卡牌风格 UI，毛玻璃效果（glassmorphism）
- 宠物以浮动 GIF 形式展示，带稀有度彩色辉光

## Data Model

### Pet (Backend, JSON)
```json
{
  "id": 1,
  "name": "布伊",
  "rarity": "common",
  "gif_url": "/assets/pets/buyi.gif",
  "description": "喜欢温暖水域的治愈系宠物"
}
```

### Database Schema (SQLite)
```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE collection (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  pet_id INTEGER NOT NULL,
  count INTEGER DEFAULT 1,
  first_obtained_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE TABLE daily_pulls (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  pull_date TEXT NOT NULL,  -- UTC date string YYYY-MM-DD
  pull_count INTEGER DEFAULT 0,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## Pull Algorithm

```
1. Generate random number N ∈ [0, 100)
2. If N < 10 → legendary (10%)
   Else if N < 40 → rare (30%)
   Else → common (60%)
3. Select random pet of the chosen rarity
4. Return pet data to frontend
```

## 移动端策略

- 粒子数量根据设备降低（桌面 400，移动端 150）
- 触摸事件支持（点击 = 触摸）
- 使用 `window.innerWidth` 检测屏幕尺寸，调整 3D 相机距离
- Three.js 渲染启用 `pixelRatio` 自适应（上限 2x）
- `prefers-reduced-motion` 检测：用户开启省动模式时，跳过 3D 动画直接显示结果

## 可访问性

- `prefers-reduced-motion`：省动模式下跳过动画直接抽卡
- 键盘支持：Tab 导航 + Enter 抽卡
- 抽卡结果同时有文字提示（ARIA live region）
